# Проект

Telegram-бот для рейтинговых игр в настольный теннис внутри закрытой группы.
Регистрация только по инвайт-коду. Один разработчик. Стадия: MVP в проде на VPS (FirstVDS).

**Что умеет бот:**
- Регистрация игроков по инвайт-ссылке (`/start <INVITE_CODE>`)
- Вызов соперника на матч (матч начинается сразу, без шага принятия)
- Внесение результата: пошаговый FSM-ввод или прямой ввод счёта в чат (`11:7 9:11 11:5`)
- Отмена принятого матча любым из участников
- Автопересчёт рейтинга после каждого матча
- Таблица рейтинга (с ▲▼ за неделю), личная статистика, история матчей, личные встречи (H2H)
- Напоминание о незавершённых матчах через 24 часа (APScheduler)
- Итоги дня каждый вечер в 21:30 МСК (топ дня + «матч дня»)
- Еженедельный дайджест каждый понедельник в 9:00 МСК

# Бизнес-правила (не нарушать)

- **Результат вносит только победитель.** Если репортёр ввёл счёт, где проиграл — отклоняется.
- **Рейтинг никогда не сбрасывается.** Только накапливается через `+delta` / `-delta` после каждого матча. Пол рейтинга: **1000.0** для новичков (до 15 матчей), **900.0** для ветеранов (15+ матчей). Стартовый рейтинг: 1000.0.
- **Рейтинг отображается с одним знаком после запятой** везде (не `int`, а `round(x, 1)`).
- **Счёт партии валидируется** по правилам настольного тенниса: победитель ≥11 очков с отрывом ≥2. Дьюс: 12:10, 13:11 и т.д.
- **Дубли матчей заблокированы**: нельзя вызвать игрока, с которым уже есть активный вызов или матч.
- **Активный матч можно отменить** любым из двух участников — оба получают уведомление.
- **MatchStatus**: `accepted` → `completed` (или `declined` при отмене). `pending` сохранён в enum только для совместимости со старыми записями БД — новые матчи создаются сразу как `accepted`.

# Рейтинговая формула

```
E                = 1 / (1 + 10 ^ ((loser_rating - winner_rating) / 400))
base_delta       = 32 * (1 - E)

sets_ratio       = winner_sets_won / total_sets        # 0.6..1.0
pts_ratio        = winner_total_pts / all_pts          # ~0.5..0.85
score_mult       = 1 + 0.5*(sets_ratio - 0.5) + 0.3*(pts_ratio - 0.5)

short_match_mult = 0.75 если len(sets_data) == 1, иначе 1.0   # штраф за 1-партийный матч
newcomer_bonus   = 1.2  если у победителя < 15 матчей, иначе 1.0
repeat_mult      = max(0.5, 1.0 - 0.05 × streak)              # streak — подряд побед над тем же соперником

delta = round(base_delta × score_mult × short_match_mult × newcomer_bonus × repeat_mult, 1)
```

Победитель: `+delta`. Проигравший: `-delta`. Пол рейтинга: **1000.0** (новичок, < 15 матчей) / **900.0** (ветеран).

# Архитектурные решения

- **MemoryStorage для FSM** — сознательный выбор для MVP. При рестарте бота незавершённые вводы результата сбрасываются. Приемлемо для малой группы.
- **DatabaseMiddleware делает commit после каждого хендлера** — хендлеры тоже делают явный commit там, где нужно промежуточное сохранение. Двойной commit безопасен с SQLAlchemy.
- **SQLite в проде** — для небольшой закрытой группы достаточно. При росте нагрузки — миграция на PostgreSQL.
- **Инвайт-код через deep-link** — `/start CODE`. Если `INVITE_CODE` не задан в `.env`, регистрация открыта для всех.

- Все сообщения бота — на **русском языке**, HTML parse_mode

# Аудитория и поведение пользователей

- **5 активных игроков**, все в одном офисе
- **3–5 матчей в день** на человека, стол один — матчи строго последовательные
- **Вызов сначала голосом** («будешь играть?»), потом через бота — поэтому напоминания о матчах и вызовах практически не нужны
- **Реванш** бывает, но нечасто
- **Споров о счёте не бывает** — доверяют друг другу
- **Лидерборд смотрят просто для интереса**, не только перед вызовом
- **Только рабочее время**, вечера и выходные почти не играют
- **Сезонов и турниров нет** — вечный рейтинг, сбрасывать не нужно
- **Еженедельный дайджест** — реакция пока неизвестна
