# Проект

Telegram-бот для рейтинговых игр в настольный теннис внутри закрытой группы.
Регистрация только по инвайт-коду. Один разработчик. Стадия: MVP в проде на VPS (FirstVDS).

**Что умеет бот:**
- Регистрация игроков по инвайт-ссылке (`/start <INVITE_CODE>`)
- Вызов соперника на матч (матч начинается сразу, без шага принятия)
- Внесение результата: пошаговый FSM-ввод или прямой ввод счёта в чат (`11:7 9:11 11:5`)
- Отмена принятого матча любым из участников
- Автопересчёт рейтинга после каждого матча
- Таблица рейтинга (с ▲▼ за неделю), личная статистика, история матчей, личные встречи (H2H)
- Достижения (27 ачивок) + пасхалки после матчей + прогресс до ближайшей счётной ачивки в статистике
- График динамики рейтинга (quickchart.io)
- Рекорды клуба и матрица доминирования (кнопки на экране рейтинга)
- Экран «Мои матчи»: активные матчи клуба + «С кем сыграть?» с рекомендациями (`get_rec_signal`)
- Напоминание о незавершённых матчах через 24 часа (APScheduler)
- Итоги дня каждый вечер в 21:30 МСК (топ дня + «матч дня»)
- Еженедельный дайджест каждый понедельник в 9:00 МСК + итоги месяца 1-го числа в 10:00 МСК
- Еженедельный offsite-бэкап БД админу в личку (понедельник 9:30 МСК)

# Бизнес-правила (не нарушать)

- **Результат может внести любой участник матча.** Если репортёр ввёл счёт, где он проиграл, бот инвертирует перспективу и корректно записывает победу соперника (`sets_data` всегда хранится в перспективе победителя; при ничьей — в перспективе challenger).
- **Рейтинг никогда не сбрасывается.** Только накапливается через `+delta` / `-delta` после каждого матча. Пол рейтинга: **1000.0** для новичков (до 15 матчей), **900.0** для ветеранов (15+ матчей). Стартовый рейтинг: 1000.0.
- **Рейтинг отображается с одним знаком после запятой** везде (не `int`, а `round(x, 1)`).
- **Счёт партии валидируется** по правилам настольного тенниса: победитель ≥11 очков с отрывом ≥2. Дьюс: 12:10, 13:11 и т.д.
- **Дубли матчей заблокированы**: нельзя вызвать игрока, с которым уже есть активный вызов или матч.
- **Активный матч можно отменить** любым из двух участников — оба получают уведомление.
- **MatchStatus**: `accepted` → `completed` (или `declined` при отмене). `pending` сохранён в enum только для совместимости со старыми записями БД — новые матчи создаются сразу как `accepted`.
- **Смена статуса матча — только атомарным CAS-UPDATE** (`UPDATE ... WHERE status='accepted'` + проверка `rowcount`) — защита от гонок: двойной тап «Всё верно», отмена vs внесение результата. После CAS текущий матч уже `completed` — в подсчётах завершённых матчей игрока исключать его через `Match.id != match_id`.
- **Граница «сегодня» — полночь по МСК** через `msk_day_start()` из `utils.py` (не UTC) — единая для экрана «Сегодня», итогов дня, пасхалок **и дневных ачивок** («Маньячелло», «Неистого» — с v2.58.0).
- **Конфиг читается из env на уровне модулей** (start.py, admin.py, database.py), поэтому `load_dotenv()` в main.py обязан стоять ДО импортов бота (ruff: per-file-ignore E402). Целочисленные env-переменные читать только через `env_int()` из utils — `int(os.getenv(...))` падает на пустом значении.
- **Игроки с 0 сыгранных матчей скрыты из рейтинг-таблицы и матрицы доминирования** (с v2.59.0) — пустые регистрации не засоряют рейтинг. На экране «С кем сыграть?» такой игрок остаётся видимым и вызываемым с пометкой «🆕 ещё не играл» — иначе новичок не смог бы получить первый матч.
- **График динамики рейтинга** доступен для себя (`rating_chart`) и для любого игрока из его профиля (`player_chart_{id}`) — общий хелпер `_send_rating_chart` в history.py. Предыдущий график в чате удаляется при построении нового (последний message_id хранится в памяти по chat_id, `_last_chart_msg`).
- **Мемная фраза под прогнозом** на экране матча (`match_phrase` в utils.py) подбирается по шансам СМОТРЯЩЕГО: фаворит (>65%), равны (35–65%), андердог (<35%). Стабильна в рамках матча — индекс по `match.id`, чтобы не «прыгала» при перерисовке. Пулы фраз: `FAVORITE_PHRASES` / `EVEN_PHRASES` / `UNDERDOG_PHRASES`.
- **«Дерби клуба»** в рекордах клуба — самая играющая пара (по числу очных завершённых матчей, от 2+).
- **Рекорды клуба** (`show_club_records`) кроме дерби: «Высший рейтинг в истории» (max `peak_rating` среди игравших), «Нагибатор клуба» (самое одностороннее очное противостояние — доминирующий выиграл ≥3 раза и ведёт по счёту, ничьи не в счёт), «В ударе сейчас» (текущая активная серия побед, от 2 подряд — отличать от «Лучшая серия побед» = исторический максимум), длинные матчи, апсеты, эпичный матч.
- **`peak_rating` обновляется при любом росте рейтинга** — и при победе, и **при ничьей** (андердог может вырасти на ничьей; до v2.61.0 ветка ничьей пик не обновляла — баг занижал рекорд «Высший рейтинг в истории»). Питает этот рекорд. Стартовое значение = рейтинг при регистрации.
- **Экран «Сегодня»** (`show_today_stats`) — общий счёт дня по всем игрокам + личный мини-итог зрителя (его W–L–D и дельта рейтинга за день). **Личная дельта считается только по матчам с участием зрителя** — `match_rating_delta` для не-участника возвращает `-rating_change`, суммировать по всем матчам дня нельзя.
- **Итоги дня** (`send_daily_summary`, 21:30 МСК) — broadcast всем. «Топ дня» с полоской формы (исходы последних 7 матчей квадратами 🟩🟥🟨), «Лучший рост»/«Отрицательный рост», «Нагибатель дня» (макс серия побед за день), «Чаще всего самбовались» (самая играющая пара за день), «матч дня», и общий **«Матчи дня»** — лог всех матчей со счётом (нейтрально, победитель жирным).
- **Итоги недели** (`send_weekly_digest`, пн 9:00 МСК) — персональное сообщение: личная шапка (твой счёт за неделю, рейтинг, место) + общий клубный блок «Топ недели» (стендинг, **вместо** прежнего длинного списка матчей) + «Герои недели» + «Матч недели». **Итоги месяца** (`send_monthly_summary`) — аналогично, club-wide.
- **Единые названия метрик** во всех сводках: пара «📈 Лучший рост» / «📉 Отрицательный рост» (день/неделя/месяц). «Чаще всего самбовались» (дерби периода) и «Нагибатель периода» считаются общими хелперами `_most_played_pair` / `_longest_streak` в scheduler.py.
- **Таймзона cron-задач задаётся явно** (`timezone="Europe/Moscow"` в каждом `CronTrigger`, часы — прямо в МСК). `CronTrigger` без аргумента `timezone` берёт ЛОКАЛЬНУЮ tz сервера (`get_localzone()`), а НЕ `timezone` у `AsyncIOScheduler` (тот действует только на триггеры из строки `'cron'`). На сервере в зоне Europe/Amsterdam расписание уезжало на 2 часа (фикс v2.63.1). Матрица доминирования (`<pre>` + номера игроков вместо кириллических имён) — кириллица в моноширинном блоке на мобильном Telegram не моноширинна и разъезжает столбцы.

# Рейтинговая формула

```
E                = 1 / (1 + 10 ^ ((loser_rating - winner_rating) / 400))
base_delta       = 32 * (1 - E)

sets_ratio       = winner_sets_won / total_sets        # 0.6..1.0
pts_ratio        = winner_total_pts / all_pts          # ~0.5..0.85
score_mult       = 1 + 0.5*(sets_ratio - 0.5) + 0.3*(pts_ratio - 0.5)

short_match_mult = 0.75 если len(sets_data) == 1, иначе 1.0   # штраф за 1-партийный матч
newcomer_bonus   = 1.2  если у победителя < 15 матчей, иначе 1.0
repeat_mult      = max(0.5, 1.0 - 0.05 × streak)              # streak — подряд побед над тем же соперником

delta = round(base_delta × score_mult × short_match_mult × newcomer_bonus × repeat_mult, 1)
```

Победитель: `+delta`. Проигравший: `-delta`. Пол рейтинга: **1000.0** (новичок, < 15 матчей) / **900.0** (ветеран).

# Архитектурные решения

- **MemoryStorage для FSM** — сознательный выбор для MVP. При рестарте бота незавершённые вводы результата сбрасываются. Приемлемо для малой группы.
- **DatabaseMiddleware делает commit после каждого хендлера** — хендлеры тоже делают явный commit там, где нужно промежуточное сохранение. Двойной commit безопасен с SQLAlchemy.
- **SQLite в проде** — для небольшой закрытой группы достаточно. При росте нагрузки — миграция на PostgreSQL.
- **Инвайт-код через deep-link** — `/start CODE`. Если `INVITE_CODE` не задан в `.env`, регистрация открыта для всех.

# Аудитория и поведение пользователей

- **5 активных игроков**, все в одном офисе
- **3–5 матчей в день** на человека, стол один — матчи строго последовательные
- **Вызов сначала голосом** («будешь играть?»), потом через бота — поэтому напоминания о матчах и вызовах практически не нужны
- **Реванш** бывает, но нечасто
- **Споров о счёте не бывает** — доверяют друг другу
- **Лидерборд смотрят просто для интереса**, не только перед вызовом
- **Только рабочее время**, вечера и выходные почти не играют
- **Сезонов и турниров нет** — вечный рейтинг, сбрасывать не нужно
- **Еженедельный дайджест** — реакция пока неизвестна
