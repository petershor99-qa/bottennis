# Анализ рейтинговой системы bottennis

> Данные на момент анализа — 63 завершённых матча, 8 игроков. Имена игроков обезличены (Игрок A–F).

---

## Как работает система сейчас

### 1. Базовый ELO

```
E           = 1 / (1 + 10 ^ ((rating_соперника - rating_победителя) / 400))
base_delta  = 32 * (1 - E)
```

Чем слабее ты относительно соперника — тем выше `base_delta`. При равных рейтингах `base_delta = 16.0`.

| Разрыв (ты слабее) | base_delta |
|---|---|
| 0 pts (равные) | 16.0 |
| −50 pts | 18.3 |
| −100 pts | 20.5 |
| −200 pts | 24.3 |

### 2. Множитель за счёт матча

```
sets_ratio  = партий_победителя / всего_партий
pts_ratio   = очков_победителя  / всего_очков

multiplier  = 1 + 0.5 × (sets_ratio − 0.5) + 0.3 × (pts_ratio − 0.5)
```

Диапазон multiplier: от **~1.05** (3:2 в упор) до **~1.40** (3:0, 11:0 в каждой).

| Формат | Примерный счёт | multiplier | Δ при равных |
|---|---|---|---|
| 3-0, разгром | 11:3 каждая | 1.34 | +21.4 |
| 3-0, уверенно | 11:7 каждая | 1.28 | +20.5 |
| 3-1 | 11:7, 11:7, 7:11, 11:7 | 1.14 | +18.3 |
| 3-2, нервно | 11:9 все | 1.05 | +16.8 |

### 3. Множитель за повтор соперника (repeat penalty)

Если ты **подряд** бьёшь одного и того же человека — дельта режется:

```
repeat_multiplier = max(0.1,  1.1 − 0.1 × (streak + 1))
```

| Подряд побед vs одного | ×mult | Δ (равные, 3-0 нормально) |
|---|---|---|
| 1-я победа | ×1.00 | +20.5 |
| 2-я подряд | ×0.90 | +18.5 |
| 3-я подряд | ×0.80 | +16.4 |
| 5-я подряд | ×0.60 | +12.3 |
| 9-я+ подряд | ×0.10 | +2.1 |

### 4. Бонус новичка

Если у победителя **меньше 15 матчей** — дельта ×1.2. Стабилизирует рейтинг на старте.

### 5. Итоговая формула

```
delta = calculate_elo(winner_rating, loser_rating, sets_data)
      × newcomer_bonus
      × repeat_multiplier

winner.rating += delta
loser.rating  -= delta   # но не ниже 1000 (новичок) / 900 (ветеран)
```

## Проблемы

### ❗ Проблема 1: одна партия даёт столько же очков, что и три

38% всех матчей (24 из 63) — это матчи из **одной партии**. Формула не умеет их отличать от полноценного матча: для 1-0 `sets_ratio = 1.0`, как у идеального 3:0.

**Результат:** одна партия даёт *больше* очков, чем три:

| Формат | Ср. Δ по БД |
|---|---|
| **1-0** (одна партия) | **20.3** |
| 2-0 | 21.4 |
| **3-0** (три партии) | **15.6** |

Топ начислений — почти все однопартийные матчи:
```
+27.6  Игрок D → Игрок B  [11:9]    ← 1 партия
+26.3  Игрок D → Игрок E  [11:7]    ← 1 партия
+25.4  Игрок F → Игрок D  [11:5]    ← 1 партия
```

### ❗ Проблема 2: repeat penalty слишком жёсткий для малой группы

При 5–6 активных игроках невозможно не повторять соперников. Лидер (Игрок A, 19 побед) регулярно сталкивается с теми же людьми и получает:

```
+9.5   Игрок A → Игрок E  [13:11  11:7  11:8]    уверенная 3-0, но streak=6+
+10.0  Игрок A → Игрок D  [11:8]
```

Минимум ×0.1 (−90% от дельты) неоправданно жёсток — игрок объективно сильнее, просто некого больше бить.

## Предложения

### ✅ Предложение 1: penalty за однопартийный матч

Добавить коэффициент `short_match_penalty = 0.75` если в матче сыграна **ровно 1 партия**:

```python
short_match_mult = 0.75 if len(sets_data) == 1 else 1.0
delta = base_delta × score_multiplier × short_match_mult × repeat_mult × newcomer_bonus
```

**Эффект:** однопартийный матч будет давать ~75% от обычного.

| Матч | Сейчас | После фикса |
|---|---|---|
| 1-0, 11:9, равные | +20.2 | +15.2 |
| 1-0, 11:3, равные | +21.4 | +16.1 |
| 2-0, 11:7 каждая, равные | +20.5 | +20.5 *(без изменений)* |

Исторические рейтинги **не пересчитываются** — изменение применяется только к новым матчам.

---

### ✅ Предложение 2: смягчить repeat penalty

Текущая формула: `max(0.1, 1.1 − 0.1 × streak)`

Предлагаемая: `max(0.5, 1.0 − 0.05 × streak)`

| Подряд | Сейчас | Предложение |
|---|---|---|
| 1x | ×1.00 | ×1.00 |
| 2x | ×0.90 | ×0.95 |
| 4x | ×0.70 | ×0.85 |
| 6x | ×0.50 | ×0.75 |
| 9x | ×0.10 | ×0.60 |
| 10x+ | ×0.10 | ×0.50 (минимум) |

Деградация плавнее, минимум 50% вместо 10%.

---

### ℹ️ Что менять не нужно

- **ELO-база** — работает корректно, правильно награждает победы над более сильными.
- **Общий разброс рейтингов** (~280 pts за 63 матча) — в норме.
- **Бонус новичка** — справляется со своей задачей.
- **Исторические рейтинги** — не пересчитывать (любое изменение формулы применяется только вперёд).
